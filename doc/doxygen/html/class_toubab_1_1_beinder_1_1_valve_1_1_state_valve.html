Detailed Description

Valve for state-bearing bindables

State-bearing bindables are properties.

Properties have a state, and can produce events when their value changes.

Properties can also consume those events from other properties, for example by setting this property to the changed value announced by the other propety.

The difference between the StateValve and the BroadcastValve is that (a) consumed events result in a state change, and (b) not every consumed event has effect: only if it may cause a state change, an event from one of the bindables in the valve is propagated to the other bindables in the valve.

Member Function Documentation

bool Toubab.Beinder.Valve.StateValve.Activate ( object  toActivate)

Activate one of the objects for which there is a bindable in the valve.

"Activate" in this context is "push the value of my property to the bound property on other objects in the valve".

Example if property X of objects A and B are in a valve, and valve.Activate(A) is called, this means the value in A.X whill be transferred to B.X.

Parameters
toActivateThe object in the valve to activate. If the object is not participating in the valve, nothing happens.
