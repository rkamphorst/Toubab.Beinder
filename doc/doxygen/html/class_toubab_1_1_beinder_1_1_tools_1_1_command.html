Detailed Description

Easy-to-use implementation of ICommand

Use one of the constructors to provide call-backs for Execute and CanExecute, and optionally one or more notify objects.

If you do not supply a callback for CanExecute, true is assumed as a default return value for any call of CanExecute.

This Command makes sure that it is only ever executing once at a time. Execute is protected by a mutex; and during a call, CanExecute will return false, regardless of whether the supplied callback returns true or not.

The notify objects are used to raise the CanExecuteChanged event, using either the INotifyPropertyChanged event, or the INotifyCollectionChanged event on the notify object(s). If notify object(s) do not implement one of these interfaces, they have no effect.

If you do not supply notify objects, the Target of the CanExecute callback is automatically configured to be the notify object.

async void Toubab.Beinder.Tools.Command.Execute ( object  parameter)

Execute command with specified parameter.

Executes the command asynchronously, and does not allow this command to execute again until the execution has ended.

If you nevertheless call this method while another execution is in progress, the execution will take place after the current execution.

While there is an execution in process, CanExecute will return false.
