Detailed Description

Represents a bindable property, event or method

A bindable can be a C# property, event or method; but it can also be anything else that has a name and is associated to an object, and can act as a property, event or method. For example, it can be a dictionary entry or a named child view.

The object that holds the property can be retrieved through Object, and can be set with IMixin.SetObject.

The bindable has a name, which is represented by the Path.

The bindable has zero or more ValueTypes that it can produce (if it is an event), consume (event handler / method) or store (property). These three kinds of bindables have corresponding interface that inherit from this one: IEvent, IEventHandler and IProperty.
