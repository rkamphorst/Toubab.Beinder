Detailed Description

Bindable property

A property stores a state, represented by Values.

When this state changes, it can raise the IEvent.Broadcast event;

When another property (or event) raises the IEvent.Broadcast event, this IProperty handle that broadcast with IEventHandler.TryHandleBroadcast and update Values.

Hence, the IProperty aggregates the IEvent and IEventHandler interfaces, and adds a property Values to store the state in.

Note that an IProperty implementation should only raise the IEvent.Broadcast event.

Property Documentation

object [] Toubab.Beinder.Bindable.IProperty.Values
get

Value(s) of the property.

A property wil generally store only one value. The reason that Values can contain zero or more values (it being an array) is to be consistent with the other bindables (IEvent, IEventHandler) which can broadcast or handle zero or more values.
