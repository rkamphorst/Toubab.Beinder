Detailed Description

Interface for a mix-in

A mix-in is a class that defines methods, events and properties that are "mixed in" with another class that it is a mix-in for.

In other words, a mix-in is a way of separately specifying extensions for another class.

In the context of Beinder, the Scanner.MixinScanner class can scan for mix-ins (actually, it scans vor Mixin{T} implementations). The mix-ins are themselves scanned as if they were normal objects; the Scanner.MixinScanner then transforms the resulting Bindable.IBindable objects into bindables that "pretend" to live on the type the mix-ins are for.

void Toubab.Beinder.Mixin.IMixin.SetObject ( object  newObject)

Set the object that this IMixin is attached to.

An mixin is "detachted" if the associated object is null, otherwise it is "attached".

Whenever a non-null object is set with this method, event handlers might be attached to that object. For that reason, make sure a null object is set whenever possible to avoid obsolete event subscriptions (that e.g. prevent an object from being collected by the garbage collector).
